## Threat Model
<threat_model>
- No external unvalidated input rendered as HTML (toast messages are plain text, rendering uses Vue's `{{}}` syntax which escapes).
- File size limit checks (15MB) exist client-side preventing browser memory overflow.
</threat_model>

## Verification
<must_haves>
- `useToast` is available globally and auto-removes toasts.
- `ToastNotification` overlays correctly and handles varying states (success, error, info).
- `FileDropzone` rejects files > 15MB.
- `ConversionProgress` updates visually based on percentage.
</must_haves>
