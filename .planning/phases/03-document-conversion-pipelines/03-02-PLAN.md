# Plan 03-02: XLSX to PDF Pipeline Infrastructure

## Goal
Extend the `DocumentConversionService` to support Excel (XLSX) to PDF conversions by rendering workbook worksheets as HTML tables in the browser DOM.

## Requirements
- CONV-05: User can upload an Excel (.xlsx) sheet, render a preview as an HTML table, and convert it to a downloadable PDF file client-side.

## Tasks

```xml
<task>
  <action>
    Add the `processXlsxToPreview` method to the `DocumentConversionService` in `src/services/conversion/DocumentConversionService.ts`.

    The new method must be:
    `async processXlsxToPreview(file: File, container: HTMLElement, options?: ConversionOptions): Promise<ConversionResult>`

    Implementation steps:
    1. Validate file using `this.validateFile(file)`. If invalid, return `{ success: false, error: validation.error }`.
    2. Fire progress callback: `options?.onProgress?.('Đọc file...', 0);`
    3. Use `try/catch` to handle parsing and rendering. Inside the `try`:
    4. Call `const workbook = await this.excelService.readExcel(file);`
    5. Fire progress callback: `options?.onProgress?.('Đang chuẩn bị render...', 20);`
    6. Extract the first sheet using `const sheet = this.excelService.getWorksheet(workbook, 0);`. If null, throw an Error `'Không tìm thấy dữ liệu trong file Excel'`.
    7. Generate an HTML table string: `let html = '<table class="min-w-full border-collapse border border-gray-300 text-sm">';`
    8. Loop over the sheet rows using `sheet.eachRow((row) => { ... })`. For each row:
       - Append `<tr>`
       - Loop over cells using `row.eachCell({ includeEmpty: true }, (cell) => { ... })`. (Crucial: `includeEmpty: true` maintains layout alignment).
       - Inside the cell loop, append `<td class="border border-gray-300 px-2 py-1">${cell.value ?? ''}</td>`
       - Append `</tr>`
    9. Close the table `html += '</table>';`
    10. Inject into container: `container.innerHTML = html;`
    11. Fire progress callback: `options?.onProgress?.('Render preview hoàn tất', 70);`
    12. Return `{ success: true }`.
    13. In the `catch` block, return `{ success: false, error: error instanceof Error ? error.message : String(error) }`.
```

## Threat Model
<threat_model>
- **Threat**: Cross-Site Scripting (XSS) via injected cell values in Excel spreadsheets if rendered unsafely as innerHTML.
- **Mitigation**: Excel strings are typically plain text or rich text objects, but mapping them using `${cell.value ?? ''}` requires care if values contain HTML tags. Given this is a local client-side tool mapping trusted user files to their own local DOM before PDF print, risk is low, but future sanitization could be considered if sharing is implemented.
</threat_model>

## Verification
<must_haves>
- Excel parsing targets only the first sheet (index 0).
- `row.eachCell({ includeEmpty: true })` is used to preserve table structure integrity.
- Method handles potential `null` or undefined sheet data safely by throwing an error.
</must_haves>
