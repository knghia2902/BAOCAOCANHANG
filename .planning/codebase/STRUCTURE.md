## Directory Purposes

**src/components/tools/**
- Purpose: House the interactive widgets for Excel tools.
- Contains: `ExcelMerger.vue` and `FormatConverter.vue`.

**src/services/excel/**
- Purpose: Execute spreadsheet extraction, conversion, formatting, and downloads.
- Contains: `ConverterService.ts`, `ExcelService.ts`, and `MergerService.ts` utilizing `exceljs`.

**src/services/storage/**
- Purpose: Storage management, auth database lookup, and local DB operations.
- Contains:
  - `AuthService.ts` - Validates logins against database config settings.
  - `DBContext.ts` - Local IndexedDB database manager.
  - `StorageService.ts` - Interface for uploading assets to Supabase Storage.

**src/stores/**
- Purpose: Maintain central reactive state variables for the frontend.
- Contains: `auth.ts` (authentication state) and `content.ts` (portfolio configurations, projects, messages).

**src/views/**
- Purpose: Individual routed views corresponding to client routes.
- Contains: Page layouts (`HomeView.vue`, `ProjectsView.vue`, `ToolsView.vue`, `ContactView.vue`, `LoginView.vue`, `AdminView.vue`, `ChangePasswordView.vue`).

## Key File Locations

**Entry Points:**
- `src/main.ts` - Client bootstrap.
- `index.html` - Base HTML container.

**Configuration:**
- `package.json` - Dependencies and build scripts.
- `.env` - Supabase keys.
- `src/router.ts` - URL routes and guard logic.
- `src/supabase.ts` - Supabase client setup.

**Core Logic:**
- `src/services/ContentService.ts` - Main data fetcher.

## Naming Conventions

**Files:**
- `PascalCase.vue`: Vue component and view templates (e.g. `ExcelMerger.vue`, `HomeView.vue`).
- `PascalCase.ts`: Service classes and namespace managers (e.g. `ConverterService.ts`).
- `camelCase.ts`: Route definitions, stores, and config wrappers (e.g. `router.ts`, `auth.ts`).

**Directories:**
- `camelCase`: Feature directories (e.g. `components/tools`, `services/excel`).
- Plural nouns: For group collections (e.g., `components/`, `services/`, `views/`).

## Where to Add New Code

**New Spreadsheet Tool:**
- Interface component: `src/components/tools/[ToolName].vue`
- Behind-the-scenes logic: `src/services/excel/[ToolName]Service.ts`
- Render entry: Import in `src/views/ToolsView.vue`

**New Routed View:**
- Create `src/views/[ViewName]View.vue`.
- Register the view and path in `src/router.ts`.

**Shared Local Logic:**
- Service actions: `src/services/[Feature]Service.ts`.
- Store variables: `src/stores/[feature].ts`.

## Special Directories

**dist/**
- Purpose: Compiled production web app assets.
- Source: Auto-generated by running `npm run build` or `vite build`.
- Committed: No (ignored via `.gitignore`).
